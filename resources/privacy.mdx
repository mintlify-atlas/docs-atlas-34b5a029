## Overview

This privacy policy applies to the BBPlayer mobile application ("Application"), created by Roitium ("Service Provider") as an Open Source service. The Application is provided "AS IS" and is designed with privacy in mind.

<Info>
  BBPlayer is committed to transparency. All data collection can be disabled through in-app settings, and the source code is publicly available for review.
</Info>

## Information Collection and Use

The Application collects limited information when you download and use it to improve the user experience and fix bugs.

### Automatically Collected Information

The following technical data may be collected:

- Your device's Internet Protocol address (IP address)
- Application pages visited, time and date of visits, and time spent
- Time spent using the Application
- Operating system information

<Note>
  The Application does **not** gather precise location information about your mobile device.
</Note>

### Location Data Usage

The Application collects approximate geographical location data for:

- **Geolocation Services**: Providing personalized content and relevant recommendations
- **Analytics and Improvements**: Analyzing user behavior patterns to improve app performance (aggregated and anonymized)
- **Third-Party Services**: Transmitting anonymized location data to external services for app optimization

## Third-Party Services

BBPlayer integrates the following third-party services for analytics and error tracking:

<Accordion title="Firebase Analytics">
  Firebase Analytics is used to understand feature usage patterns and user engagement. This helps prioritize development efforts and improve the overall experience.
  
  **What it collects:**
  - Feature usage frequency
  - Session duration and patterns
  - Anonymous device and OS information
  
  **Privacy Policy:** [Google / Firebase Privacy](https://policies.google.com/privacy)
</Accordion>

<Accordion title="Sentry">
  Sentry is used for crash reporting and error monitoring. When the app crashes, stack traces and error information are sent to help identify and fix bugs.
  
  **What it collects:**
  - Crash reports with stack traces
  - Error messages and app state at time of error
  - Device and OS information for debugging
  
  <Warning>
    Sentry reports do **not** include any account credentials, cookies, or personal viewing history.
  </Warning>
  
  **Privacy Policy:** [Sentry Privacy](https://sentry.io/privacy/)
</Accordion>

<Accordion title="Expo">
  The app is built using Expo, which provides development and runtime services.
  
  **Privacy Policy:** [Expo Privacy](https://expo.io/privacy)
</Accordion>

### Our Privacy Commitments

<Note>
  **Anonymous**: All collected data is anonymized and does not contain personally identifiable information.
  
  **Transparent**: We do not collect account credentials, cookies, or detailed browsing history. All analytics code is open source and publicly viewable.
  
  **User Control**: You can disable all data collection at any time through the in-app settings.
</Note>

## In-App Opt-Out

You have full control over data collection:

1. Open **Settings** in the app
2. Navigate to **General** (通用设置)
3. Toggle **"Share Data (Crash Reports & Anonymous Stats)"** (分享数据)

<Info>
  Disabling this setting will completely stop all anonymous data collection, including crash reports and analytics.
</Info>

The Service Provider may disclose user-provided and automatically collected information:
